# 요구사항 정의서

## 1. 개요 (Introduction)

### 1.1 프로젝트 목적

- 안정적인 IBT(Internet Based Test) 환경 구축을 통한 온라인 평가의 공정성 및 효율성 확보함.
- 강화된 보안 스크립트를 통해 부정행위를 원천 차단하고 관리자의 실시간 관제 능력을 극대화함.

### 1.2 시스템 범위

- **관리자 시스템:** 문항 제작, 시험장 개설, 초대코드 배포, 실시간 모니터링, 성적 분석함.
- **피시험자 시스템:** 환경 점검, 보안 응시 엔진, 답안 자동 저장함.

## 2. 기능적 요구사항 (Functional Requirements)

### 2.1 문항 및 문제 은행 관리 (Question Bank)

- **FE-01 (유형별 에디터):** 객관식, 주관식(단답/서술), 빈칸 채우기 문항을 작성할 수 있는 리치 텍스트 에디터 제공함.
- **FE-02 (이미지 관리):** 문항 및 보기 내 이미지 삽입 기능을 지원하며, 업로드된 이미지는 클라우드 스토리지(S3)와 연동함.
- **FE-03 (문항 복제):** 기존 등록된 문항의 모든 속성을 유지한 채 새로운 문항으로 즉시 복제하는 기능을 구현함.
- **FE-04 (일괄 등록):** 관리자가 정의된 Excel 템플릿을 통해 수백 개의 문항을 한 번에 업로드할 수 있어야 함.

### 2.2 시험장 및 세션 관리 (Exam Room)

- **FE-05 (시험장 개설):** 특정 시험(Exam)을 기반으로 날짜, 시작 시각, 종료 시각, 개별 응시 시간(분)을 설정한 독립된 시험장을 개설함.
- **FE-06 (초대코드 격리):** 시험장별로 독립된 초대코드 그룹을 생성하며, 타 시험장의 코드는 접근이 불가능하도록 권한을 격리함.
- **FE-07 (이메일 배포):** 관리자가 등록한 피시험자 명단을 대상으로 초대코드와 접속 URL이 포함된 안내 메일을 자동/수동 발송함.

### 2.3 응시 엔진 (Test Engine)

- **FE-08 (코드 인증):** 별도의 회원가입 없이 초대코드 입력만으로 세션에 진입하며, 이미 사용 중인 코드는 중복 접속을 차단함.
- **FE-09 (환경 점검):** 시험 시작 전 전체 화면 전환 및 시스템 사양(브라우저, 네트워크)을 강제 점검하는 단계를 거침.
- **FE-10 (효율적 자동 저장):** 답안 입력 후 3초의 디바운싱(Debounce) 주기 또는 항목 포커스 아웃(Blur) 시점에 서버로 답안을 임시 전송(Redis 캐싱 연동)하여 잦은 통신 부하를 최적화함.
- **FE-11 (타이머 제어):** 서버 시간 기준의 카운트다운을 수행하며, 제한 시간 종료 시 답안을 자동 제출하고 화면을 잠금함.

### 2.4 모니터링 및 결과 분석 (Monitoring & Analytics)

- **FE-12 (실시간 관제):** 관리자 대시보드에서 응시자별 진행률, 접속 상태, 보안 위반 횟수를 실시간 갱신(Socket)하여 노출함.
- **FE-13 (보안 로그):** 위반 발생 시점(타임라인)과 유형을 상세 기록하며, 필요시 관리자가 원격으로 세션을 강제 종료함.
- **FE-14 (성적 집계):** 객관식/단답형은 자동 채점하며, 서술형은 관리자의 수동 채점 기능을 제공함. (피시험자는 결과 조회 불가)
- **FE-15 (데이터 추출):** 성적 결과 및 응시 로그를 Excel/CSV 형태로 다운로드할 수 있어야 함.

## 3. 보안 및 비기능적 요구사항 (Security & Non-Functional)

### 3.1 부정행위 방지 (Technical Lockdown)

- **SE-01 (포커스 감지):** 브라우저 창 이탈(Blur) 이벤트를 감지하여 경고를 발부하고, 설정된 횟수(예: 3회) 초과 시 자동 실격 처리함.
- **SE-02 (전체 화면 강제):** Fullscreen API를 사용하여 전체 화면 모드를 강제하며, 해제 시 즉시 문항 노출을 중단함.
- **SE-03 (입력 제어):** 마우스 우클릭, 개발자 도구(F12), 복사/붙여넣기 단축키, 화면 캡처 키 입력을 원천 차단함.

### 3.2 안정성 및 성능 (Reliability & Performance)

- **NF-01 (대규모 트래픽 분산):** 1,000명 이상의 동시 접속 환경에서 잦은 답안 이벤트 병목을 줄이기 위해, Redis 메모리 기반 'Write-back 1차 캐싱' 후 정기적으로 RDBMS에 Bulk Upsert 처리함.
- **NF-02 (엄격한 세션 타이머 계산):** 비정상 종료 시각부터 재접속 시점까지의 차이 또한 개인의 시험 시간에 포함하여, 서버 DB 기준 `started_at + duration` 초과 여부를 무조건적으로 산정함.
- **NF-03 (데이터 무결성):** 모든 답안 데이터는 PostgreSQL의 트랜잭션을 통해 최종 저장 실패가 없도록 보장함.

## 4. 데이터 요구사항 (Data Requirements)

- **DR-01 (JSONB 활용):** 가변적인 문항 구조(객관식 보기, 빈칸 위치 등)를 처리하기 위해 비정형 데이터 타입을 적극 활용함.
- **DR-02 (타임스탬프):** 모든 위반 행위와 답안 갱신은 서버 표준시(UTC) 기준의 타임스탬프를 기록함.

## 5. 인터페이스 요구사항 (Interface Requirements)

- **IF-01 (외부 메일 연동):** AWS SES 또는 전문 SMTP 서비스를 연동하여 대량 메일 발송의 수신 성공률을 확보함.
- **IF-02 (Excel 연동):** 전용 라이브러리(ExcelJS 등)를 사용하여 관리자의 데이터 업로드/다운로드 편의성을 제공함.
