# 기능 정의 명세

## 1. 관리자 시스템 (Admin Console)

### 1.1 문항 및 문제 은행 관리 (Question Management)

- **QM-01 (멀티 에디터):** 리치 텍스트(Rich Text) 및 이미지가 포함된 문항 작성 기능을 제공함.
- **QM-02 (유형별 컴포넌트):** 객관식(단일/다중), 주관식(단답), 빈칸 채우기(문장 내 입력 필드) 폼을 지원함.
- **QM-03 (문항 복제):** 기존 문항 리스트에서 '복제' 버튼 클릭 시 동일 속성의 신규 문항을 자동 생성함.
- **QM-04 (엑셀 일괄 등록):** 표준 템플릿(Excel) 파싱을 통해 문항 대량 등록 및 유효성 검사를 수행함.
- **QM-05 (미리보기):** 저장 전 피시험자에게 노출될 실제 IBT 화면을 미리 확인하는 기능을 제공함.

### 1.2 시험 및 시험장 관리 (Exam & Room Management)

- **ER-01 (시험지 구성):** 문제 은행에서 개별 문항을 선택하여 하나의 시험지(Exam)를 패키징함.
- **ER-02 (시험장 개설):** 특정 시험지를 기반으로 날짜, 시작/종료 시각, 응시 시간(분)을 설정한 세션을 생성함.
- **ER-03 (초대코드 그룹 관리):** 시험장별 독립된 초대코드 풀(Pool)을 생성하고 접근 권한을 격리함.
- **ER-04 (운영 정책 설정):** 문항 셔플, 보안 감도(이탈 허용 횟수), 타이머 방식 등을 시험장별로 설정함.

### 1.3 응시자 관리 및 배포 (Participant & Distribution)

- **PD-01 (응시자 명단 업로드):** 이름, 이메일 등이 포함된 엑셀 파일을 업로드하여 수험생 DB를 구축함.
- **PD-02 (초대코드 자동 매칭):** 업로드된 명단과 생성된 초대코드를 1:1로 자동 매핑함.
- **PD-03 (이메일 발송 엔진):** AWS SES 연동을 통해 초대코드 및 다이렉트 응시 URL을 개별/전체 발송함.
- **PD-04 (발송 상태 추적):** 메일 발송 성공/실패 여부를 실시간으로 모니터링함.

### 1.4 실시간 관제 및 결과 (Monitoring & Analytics)

- **MA-01 (실시간 대시보드):** Socket.io를 통해 응시자별 진도율, 접속 상태, 보안 위반 현황을 실시간 중계함.
- **MA-02 (원격 제어):** 특정 응시자에 대한 경고 메시지 전송 및 강제 시험 종료 기능을 제공함.
- **MA-03 (채점 엔진):** 객관식/빈칸 채우기는 정답 매칭을 통해 시험 종료 즉시 자동 채점되며, 서술형 주관식은 관리자의 대시보드에서 수동 부분 점수 부여 프로세스를 지원함.
- **MA-04 (리포트 추출):** 전체 성적 통계 및 개인별 보안 위반 로그가 포함된 리포트를 Excel로 추출함.

## 2. 피시험자 시스템 (Taker Engine)

### 2.1 인증 및 환경 점검 (Entry & Setup)

- **TE-01 (초대코드 인증):** 고유 코드를 통한 세션 검증 및 중복 로그인 차단 로직을 수행함.
- **TE-02 (보안 샌드박스 활성화):** '시험 시작' 클릭 시 Fullscreen API를 호출하여 전체 화면 모드를 강제함.
- **TE-03 (디바이스 점검):** 브라우저 호환성, 네트워크 지연 시간, 모니터 해상도 등을 자동 체크함.

### 2.2 IBT 응시 엔진 (Core Engine)

- **CE-01 (동적 문항 렌더링):** 데이터 기반으로 객관식/주관식/빈칸 입력 필드를 동적으로 렌더링함.
- **CE-02 (실시간 답안 동기화):** 답안 마킹 시 즉각적인 호출 대신 3초 디바운싱(Debounce) 또는 포커스 아웃(Blur) 이벤트를 트리거로 삼아 서버 API를 비동기 호출함 (Redis를 거쳐 단계적 DB 반영).
- **CE-03 (개별 타이머):** 서버에서 전달된 잔여 시간 데이터를 기준으로 클라이언트 사이드 타이머를 구동함.

### 2.3 보안 제어 (Lockdown Security)

- **LS-01 (이탈 감지):** `window.blur` 및 `visibilitychange` 이벤트를 감지하여 서버로 즉시 전송함.
- **LS-02 (입력 차단):** 우클릭 메뉴, 단축키(F12, Ctrl+C/V 등)의 브라우저 기본 동작을 무효화함.
- **LS-03 (복귀 강제):** 전체 화면 해제 시 문항 오버레이(암전) 처리 및 복귀 버튼을 강제 노출함.

## 3. 공통 시스템 기능 (System Commons)

- **SC-01 (서버 시간 동기화):** 모든 비즈니스 로직(시작/종료)은 DB 서버 표준 시간을 기준으로 처리함.
- **SC-02 (세션 복구 로직):** 재접속 시 이전 응시 기록을 Redis에서 조회하여 마지막 풀이 문항부터 재개함.
- **SC-03 (에러 처리):** 네트워크 장애 시 사용자에게 알림을 띄우고 로컬 스토리지에 답안을 임시 백업함 (단, 복구 후 서버 측 잔여 시험 시간(종료 후 1분 유예 폭) 이내 전송분만 유효 데이터로 처리함).
