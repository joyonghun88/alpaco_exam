# 관리자 포털(Admin Portal) 기능 명세 및 개발 리스트

현재까지 구현된 관리자 대시보드(관제탑)는 전체 시험 플랫폼에서 가장 코어가 되는 일부(모니터링)입니다. 완전한 무감독 온라인 시험 플랫폼의 관리자 포털 구조를 완성하기 위해 필요한 전체 메뉴 구성과 우선순위 개발 리스트를 정리합니다.

---

## 1. 인증 및 권한 (Auth & Security)
대시보드와 응시자 개인정보, 시험 문항이 외부에 노출되지 않도록 강력한 보호 장치가 필요합니다.
- [ ] **관리자 로그인 페이지 (`/admin/login`)**: JWT(JSON Web Token) 혹은 세션을 활용한 관리자 전용 인증 뷰.
- [ ] **라우터 가드(Route Guard)**: 로그인하지 않은 유저가 `/admin/*` 하위 경로에 접근 시 로그인 뷰로 튕겨내는 프론트엔드 로직.
- [ ] **권한 분리 (RBAC)**: '최고 관리자(문항/시험 세팅 권한 보유)'와 '일반 감독관(실시간 관제 및 뷰어 권한만 보유)'의 권한 분리 설계.

## 2. 대시보드 및 실시간 관제 (Live Monitoring) - *현재 진행 중*
실시간으로 진행되고 있는 시험의 통계를 확인하고, 사고에 대응하는 종합 통제실입니다.
- [x] 전체 통계 요약표 (전체 인원/응시 중/완료/위반 누적 수 등)
- [x] 실시간 응시자 리스트 및 소켓(Socket.io) 기반 상태 동기화
- [ ] **상세 로그 베젤/모달**: 특정 인원 클릭 시, "14시 10분 화면 이탈", "14시 12분 전체화면 종료" 등 위반 시간 타임라인 제공.
- [ ] **강제 제어 기능**: 심각한 부정행위자 적발 시, [강제 제출 후 퇴장] 버튼 발동으로 해당 클라이언트 즉시 종료.

## 3. 시험 관리 (Exam Management)
시험 과목(분류)을 개설하고 언제, 어떤 문항으로 평가할지 세팅합니다.
- [ ] **시험 목록 조회 및 뼈대 개설**: 과목명, 설명 등을 입력하여 기초 시험(`Exam`) 생성.
- [ ] **문항 작성 및 배점 (Question Bank)**: 해당 시험에 출제될 문항(객관식, 단답형, 서술형)을 작성 및 뷰 관리.
- [ ] **고사장(Exam Room) 개설**: 생성된 시험을 바탕으로 "2026 하반기 공채장" 등 실제 응시 시간(Start/End)과 제한시간(Duration)을 담보하는 방 생성. (문제 셔플 여부 등 옵션 제어)

## 4. 수험생 관리 (Participant Management)
시험을 치를 대상자를 관리하고 통지합니다.
- [ ] **수험자 개별/일괄 등록**: 실제 엑셀 파일 등을 파싱하여 대상자(이름, 이메일) 대량 등록 기능.
- [ ] **초대코드 발급**: 등록된 인원에게 고유 `invitationCode`를 난수로 자동 매칭하는 로직.

## 5. 채점 및 결과 (Grading & Reporting)
시험 종류 후 결과를 분석하고, 성적 산출 및 내보내기를 수행합니다.
- [ ] **수동 서술형 채점 뷰**: 에세이나 서술형 문항을 열어보고 감독관이 키워드를 채점해 수기로 점수를 매기는 화면 구조.
- [ ] **보고서 요약 및 Export**: 최종 성적 결과와 위반 로그 내역을 취합하는 화면.

---

### 💡 [제안하는 다음 단계 개발 순서]
위 기능들을 바탕으로 한 개발 추천 순서입니다.

1. **글로벌 사이드바(GNB) 레이아웃 구성**: Dashboard 위에 좌측 또는 상단 내비게이션 바를 달아 각 메뉴로 이동할 수 있는 기본 틀 마련.
2. **(Auth) 관리자 전용 인증망 개설**: 임시 백엔드 관리자 계정을 만들고, `/admin/login` 화면을 구성하여 JWT 기반 세션 방어 로직 완성.
3. **(CRUD) 시험지 & 고사장 세팅 메뉴**: 대시보드의 바탕이 되는 '시험 데이터'를 관리자가 직접 입력하고 수정할 수 있는 화면 작성.
