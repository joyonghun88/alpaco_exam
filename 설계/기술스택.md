# 기술스택

## 1. 프론트엔드 (Frontend) - 응시 환경 및 관리 UI

| **기술 스택** | **사용처 및 핵심 기능** |
| --- | --- |
| **React.js** | - 피시험자용 고성능 SPA(Single Page Application) 시험 화면 구축함.
- 관리자용 실시간 모니터링 대시보드 UI 구현함. |
| **Tailwind CSS** | - 다양한 디바이스(PC, 태블릿) 대응을 위한 반응형 레이아웃 설계함.
- 시험 집중도 향상을 위한 다크모드 및 미니멀한 UI 테마 적용함. |
| **Zustand** | - 가볍고 직관적인 상태 관리를 통해 실시간 답안 마킹 상태 및 남은 시간 제어함.
- 서버 연결 상태 및 보안 위반 횟수 실시간 추적함. |
| **Socket.io-client** | - 서버와의 상시 연결을 통해 보안 위반 로그(이탈 등) 즉시 전송함.
- 관리자의 실시간 공지사항 및 강제 종료 명령 수신함. |
| **ExcelJS** | - 관리자 페이지에서 엑셀 서식을 이용한 문제 일괄 입력 시 브라우저 단의 데이터 파싱 및 검증 수행함. |

## 2. 백엔드 (Backend) - 데이터 처리 및 실시간 관제

| **기술 스택** | **사용처 및 핵심 기능** |
| --- | --- |
| **NestJS (Node.js)** | - TypeScript 기반의 안정적인 API 서버 아키텍처 구축함.
- 실시간 데이터 전송 처리에 최적화된 이벤트 기반 비즈니스 로직 수행함. |
| **Socket.io** | - 수천 명의 응시자와 관리자 사이의 양방향 실시간 통신 브릿지 역할 수행함.
- 특정 응시자의 화면 이탈 시 관리자 대시보드로 즉각적인 알림 푸시함. |
| **JWT (Json Web Token)** | - 초대 코드를 기반으로 한 피시험자 세션 인증 및 보안 토큰 발급함.
- 관리자 계정의 API 접근 권한 제어함. |
| **Prisma** | - 데이터베이스 모델링 직관화 및 유연한 JSONB 타입 캡슐화로 스키마 안정성 확보함.
- 복잡한 성적 합계 및 통계 및 수동 채점용 관계 데이터를 효과적으로 추상화함. |

## 3. 데이터베이스 및 스토리지 (Storage & Cache)

| **기술 스택** | **사용처 및 핵심 기능** |
| --- | --- |
| **PostgreSQL** | - 문제 은행(텍스트), 시험 차수 정보, 수험생 명단, 최종 성적 등 핵심 관계형 데이터 저장함.
- 트랜잭션 보장을 통해 답안 제출의 무결성 확보함. |
| **Redis** | - 실시간 응시 현황 캐싱 및 다수 서버 컨테이너 간의 Socket.io 이벤트를 연결하는 Pub/Sub(Adapter) 역할 수행함.
- 시험 도중 잦은 저장 요청을 소화하는 Write-back 임시 캐시 저장소로 활용. |
| **AWS S3** | - 시험 문제에 포함된 이미지, PDF 가이드, 오디오 파일 등 정적 리소스 저장함. |

## 4. 보안 및 인프라 (Security & Infra)

| **기술 스택** | **사용처 및 핵심 기능** |
| --- | --- |
| **Full-screen API** | - 시험 시작 시 브라우저의 전체 화면 모드 강제 및 해제 이벤트 감시함. |
| **Page Visibility API** | - 응시자의 현재 탭 활성화 여부(Focus/Blur)를 정밀하게 감지함. |
| **AWS CloudFront** | - 이미지 등 시험 리소스를 CDN을 통해 전송하여 로딩 속도 극대화함. |
| **Docker / K8s** | - 환경에 구애받지 않는 애플리케이션 배포 및 시험 집중 시간에 따른 서버 수평 확장 지원함. |
